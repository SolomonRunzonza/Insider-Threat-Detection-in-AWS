# Insider-Threat-Detection-in-AWS

## Project Overview

This project demonstrates the design and implementation of an Insider Threat Detection solution using Amazon Web Services (AWS) integrated with Splunk Enterprise for centralized log management, threat monitoring, and security analytics. The objective was to detect, investigate, and respond to suspicious user activities and potential insider threats through automated monitoring, log analysis, and alerting.

## Solution Architecture

### Data Flow

1. User activities are logged through AWS CloudTrail.
2. CloudTrail logs are stored in Amazon S3.
3. AWS CloudWatch collects and monitors events.
4. Security findings are generated by GuardDuty and Security Hub.
5. Logs and security events are forwarded to Splunk.
6. Splunk indexes, correlates, and analyzes security events.
7. Custom SPL searches identify suspicious behavior.
8. Alerts are generated for security incidents and insider threat indicators.
9. SNS notifications provide near real-time alerting.

#### Sensitive Data Access Monitoring

Correlated user activities to identify potentially malicious behavior from legitimate users.

#### Threat Hunting

Performed investigations using SPL queries to identify indicators of compromise and anomalous activity.

### Example SPL Queries

#### Failed Login Attempts

```spl
index=aws sourcetype=aws:cloudtrail eventName=ConsoleLogin errorMessage=*
| stats count by userIdentity.userName sourceIPAddress
```

#### IAM Policy Changes

```spl
index=aws sourcetype=aws:cloudtrail
(eventName=AttachUserPolicy OR eventName=PutUserPolicy OR eventName=CreatePolicy)
```

#### Privileged Role Assumption

```spl
index=aws sourcetype=aws:cloudtrail eventName=AssumeRole
| stats count by userIdentity.userName requestParameters.roleArn
```
